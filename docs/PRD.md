# 产品需求文档 (PRD)
# Vibecoding 成绩统计可视化工具

## 1. 背景与目标

当前Vibecoding训练营的成绩统计主要依赖Excel表格，缺乏直观的可视化展示，学生难以快速了解自己的表现趋势，老师也难以把握整体学习情况。

本项目旨在构建一个**本地运行的Web应用**，为AI编程挑战活动提供成绩录入、统计分析和可视化展示功能，让师生都能快速掌握学习进展。

## 2. 目标用户

Vibecoding训练营的学生（查看个人成绩与趋势）和老师/助教（管理全班数据与整体分析）。

## 3. 核心功能

| 功能 | 描述 |
|------|------|
| **数据管理** | 支持手动录入学生成绩，或导入Excel/CSV文件批量导入 |
| **统计看板** | 自动计算总分、排名、完成率，生成趋势图和榜单 |
| **数据持久化** | 本地存储数据（浏览器LocalStorage），刷新不丢失 |

## 4. 非目标

- ❌ 不支持多人同时编辑（单机本地使用）
- ❌ 不提供用户账号/登录系统
- ❌ 不连接外部数据库或后端服务
- ❌ 不导出PDF/图片报告（可后续扩展）

## 5. 完成标准

- [ ] 能够手动录入至少10条学生成绩记录
- [ ] 能够导入CSV文件并正确解析数据
- [ ] 统计看板展示：总分、排名、完成率、趋势折线图
- [ ] 数据刷新页面后不丢失
- [ ] 界面简洁可用，无明显bug

## 6. 风险与降级方案

| 风险 | 降级方案 |
|------|----------|
| CSV解析复杂，时间不够 | 砍掉CSV导入，只保留手动录入 |
| 图表库学习成本高 | 砍掉趋势图，只保留表格+基础统计 |
| 界面美化耗时 | 使用现成UI组件库（如Tailwind CSS） |
| 本地存储数据丢失风险 | 增加"导出数据为JSON"功能备份 |

---

**技术选型建议**：HTML + CSS + JavaScript（原生或Vue CDN）+ Chart.js（图表）+ Tailwind CSS（样式），可本地直接打开运行，无需安装环境。
